# 项目技术说明

## 1. 项目的整体功能

本项目是一个基于PySide6的工业自动化控制系统，主要用于化学反应器及其后处理过程的自动化控制。系统通过串口和以太网通信，实现对温度、电机、阀门、泵等工业设备的集中监控与管理。

**核心功能包括：**
- 反应器模块控制（温度监测与控制、搅拌电机控制）
- 加液模块控制（定量泵、进液阀门、通氮气阀门）
- 后处理模块控制（泵、开关阀、电机、气体阀门、排放阀门等）
- 三轴运动控制系统
- 单步操作与多步序列操作支持
- UDP信号接收功能（用于外部系统通信）
- 参数实时监控与UI反馈

## 2. 整个代码的架构

项目采用模块化架构设计，遵循MVC（模型-视图-控制器）模式，将业务逻辑与界面分离，提高代码的可维护性和扩展性。

**整体架构层次：**
```
┌─────────────────────────────────────────────────────────┐
│                     用户界面层 (View)                   │
│  - MainUI (主界面)                                      │
└───────────────────────────┬─────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────┐
│                  控制管理层 (Controller)                │
│  - ButtonActionManager (按钮事件管理)                   │
│  - InputActionManager (输入控制管理)                    │
│  - DisplayActionManager (显示控制管理)                  │
│  - UIFeedbackHandler (UI反馈处理)                       │
└───────────────────────────┬─────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────┐
│                  业务逻辑层 (Model)                     │
│  - DeviceManager (设备管理)                             │
│  - SingleStepActions (单步操作)                         │
│  - MultiStepActions (多步操作)                          │
│  - ParameterStorage (参数存储)                          │
└───────────────────────────┬─────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────┐
│                  设备驱动层 (Driver)                    │
│  - 串口设备驱动 (SerialDevices)                        │
│  - 以太网设备驱动 (EthernetDevices)                    │
└───────────────────────────┬─────────────────────────────┘
                            │
┌───────────────────────────▼─────────────────────────────┐
│                  外部设备层 (Devices)                   │
│  - 温度控制器、电机、阀门、泵等                          │
└─────────────────────────────────────────────────────────┘
```

**模块依赖关系：**
- MainUI 依赖于 PySide6 框架，负责界面展示
- 控制管理层依赖于 MainUI 和业务逻辑层，负责用户交互处理
- 业务逻辑层依赖于设备驱动层，负责业务流程控制
- 设备驱动层依赖于硬件设备，负责设备通信与控制

## 3. 每个模块的具体功能及内容

### 3.1 UIInteraction（用户界面交互）

**功能：** 负责用户界面的生成、控制和交互处理，是系统与用户的交互桥梁。

**子模块：**

#### 3.1.1 UIGenerator（界面生成器）
- **MainUI.py**：使用PySide6构建的主界面，包含所有控件布局和初始化

#### 3.1.2 ControlActions（控制动作）
- **ButtonActionManager.py**：管理所有按钮的点击事件，连接UI操作与业务逻辑
- **InputActionManager.py**：管理输入框的验证和输入处理
- **DisplayActionManager.py**：管理界面显示内容的更新

#### 3.1.3 ParameterManagement（参数管理）
- **ParameterStorage.py**：存储和管理系统所有参数

#### 3.1.4 RealTimeUpdate（实时更新）
- **RealTimeUpdate.py**：实现参数的实时更新和显示

#### 3.1.5 UIFeedback（界面反馈）
- **UIFeedbackHandler.py**：处理UI反馈信息，如状态提示、错误警告等

### 3.2 BusinessActions（业务动作）

**功能：** 实现系统的核心业务逻辑，包括设备控制、动作执行等。

**子模块：**

#### 3.2.1 DeviceManager（设备管理）
- **DeviceManager.py**：管理所有设备实例，负责设备的连接、断开和状态监控

#### 3.2.2 SingleStepActions（单步操作）
- **MotorAction.py**：电机控制相关操作
- **TemperatureControlAction.py**：温度控制相关操作
- **ValveAction.py**：阀门控制相关操作
- **AxisSingleStepAction.py**：轴控制相关操作
- **SwitchValveAction.py**：开关阀控制相关操作

#### 3.2.3 MultiStepActions（多步操作）
- **MultiStepActionManager.py**：管理多步序列操作

### 3.3 ActionSequence（动作序列）

**功能：** 实现动作序列的执行和管理，支持复杂的多步操作流程。

**子模块：**
- **execute_sequence.py**：动作序列执行引擎
- **extract_command_from_txt.py**：从文本文件提取命令序列

### 3.4 Drivers（设备驱动）

**功能：** 实现与硬件设备的通信和控制，是业务逻辑与硬件设备之间的桥梁。

**子模块：**

#### 3.4.1 SerialDevices（串口设备）
- **Common_Serial.py**：通用串口通信类
- **TemperatureController.py**：温度控制器驱动
- **PumpController.py**：泵控制器驱动
- **SwitchValveController.py**：开关阀控制器驱动
- **Motor485_ZhangDaTou.py**：485电机驱动
- **Motor485_iDM42.py**：iDM42电机驱动

#### 3.4.2 EthernetDevices（以太网设备）
- **MotorZmc.py**：ZMC电机驱动
- **SRND_16_IO.py**：IO控制器驱动
- **Valve.py**：阀门驱动
- **ZMC.py**：正运动控制卡驱动
- **inovance_three_axis/**：汇川三轴控制器驱动

### 3.5 Common（公共模块）

**功能：** 提供系统通用的配置和工具类。

**子模块：**
- **Global.py**：全局配置和常量定义

### 3.6 DateBaseManager（数据库管理）

**功能：** 实现数据的持久化存储和管理。

**子模块：**
- **database_manager.py**：数据库连接和操作

### 3.7 UDP通信模块

**功能：** 实现UDP信号的接收，用于与外部系统通信。

**子模块：**
- **UDP_recivecmd.py**：UDP信号接收器类

## 4. 每个模块的具体技术实现细节

### 4.1 UIInteraction.UIGenerator.MainUI

**技术实现：**
- 使用PySide6框架构建Qt界面
- 采用QMainWindow作为主窗口，包含多个QTabWidget实现标签页布局
- 使用QPushButton、QLineEdit、QComboBox、QLabel等控件构建用户界面
- 界面元素通过代码动态创建和布局

**核心功能：**
- 反应器模块设置界面（电机转速、温度、加液量等）
- 后处理模块设置界面（泵参数、阀门控制等）
- 三轴位置标定界面

### 4.2 UIInteraction.ControlActions.ButtonActionManager

**技术实现：**
- 采用事件驱动模型，将按钮点击事件与业务逻辑函数绑定
- 使用lambda表达式实现参数传递
- 支持动态按钮状态管理和颜色变化

**核心功能：**
- 串口刷新和连接按钮事件处理
- 电机启动/停止按钮事件处理
- 温度设置按钮事件处理
- 阀门控制按钮事件处理
- 加液操作按钮事件处理
- 动作序列执行按钮事件处理

### 4.3 UIInteraction.ControlActions.InputActionManager

**技术实现：**
- 使用QIntValidator和QDoubleValidator实现输入验证
- 采用线程定时更新输入值
- 支持输入框的实时参数同步

**核心功能：**
- 为所有输入框设置验证规则（数值范围、小数位数等）
- 处理组合框选择事件
- 实时更新输入参数到ParameterStorage

### 4.4 BusinessActions.DeviceManager

**技术实现：**
- 采用单例模式管理设备实例
- 使用线程监控设备参数变化
- 支持设备分组管理（反应器设备、后处理设备等）

**核心功能：**
- 设备连接与断开管理
- 当前设备选择与切换
- 设备参数实时监控
- 多设备协调控制

## 5. 系统工作流程

### 5.1 系统初始化流程
1. 启动应用程序（main.py）
2. 创建QApplication实例
3. 初始化MainUI界面
4. 创建设备管理器、参数存储、UI反馈处理器等核心组件
5. 设置界面控制器（按钮、输入、显示）
6. 启动实时更新线程
7. 显示主窗口

### 5.2 设备连接流程
1. 用户选择串口或网络设备
2. 点击连接按钮
3. DeviceManager创建相应设备实例
4. 建立与硬件设备的通信
5. 更新UI显示连接状态

### 5.3 动作执行流程
1. 用户设置参数并点击执行按钮
2. ButtonActionManager捕获点击事件
3. 调用相应的业务逻辑函数（如电机启动、温度设置等）
4. 业务逻辑函数通过DeviceManager调用设备驱动
5. 设备驱动与硬件设备通信执行动作
6. 设备状态通过UIFeedbackHandler反馈到UI界面

## 6. 技术栈

- **开发语言：** Python 3.x
- **GUI框架：** PySide6
- **串口通信：** pyserial
- **网络通信：** socket（UDP）
- **数据库：** SQLite
- **多线程：** threading、QThread
- **设备驱动：** 自定义驱动（基于设备通信协议）

## 7. 系统特点

1. **模块化设计：** 系统采用模块化架构，各模块职责明确，便于维护和扩展
2. **实时监控：** 支持设备参数的实时监控和UI反馈
3. **多设备支持：** 同时支持串口和以太网设备的集中控制
4. **灵活的操作方式：** 支持单步操作和多步序列操作
5. **外部系统通信：** 支持UDP信号接收，便于与外部系统集成
6. **安全可靠：** 完善的异常处理机制，确保系统稳定运行
