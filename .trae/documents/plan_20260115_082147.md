# 状态反馈功能实现方案

## 一、需求分析

当前 `ResinWorkstation` 类已实现多种控制指令，但缺少详细的状态反馈功能。根据业务需求，需要实现以下状态反馈：

### 1. 反应器状态
- 温度（当前温度、目标温度）
- 搅拌状态（运行状态、当前转速）
- 气体状态（氮气/空气开关状态）
- 反应器运行状态（空闲/运行中/错误）

### 2. 后处理系统状态
- 清洗状态
- 排液状态
- 溶液转移状态

### 3. 溶液添加系统状态
- 溶液添加状态
- 剩余体积
- 目标体积

### 4. 设备整体状态
- 连接状态
- 操作模式
- 设备运行状态
- 错误信息
- 最后更新时间

## 二、实现方案

### 1. 定义状态数据结构
- 创建 `DeviceState` 类用于存储设备整体状态
- 创建 `ReactorState` 类用于存储单个反应器状态
- 创建 `PostProcessState` 类用于存储后处理系统状态

### 2. 实现状态查询机制
- 添加 `_get_device_state` 方法获取设备整体状态
- 添加 `_get_reactor_state` 方法获取反应器状态
- 添加 `_get_post_process_state` 方法获取后处理系统状态
- 扩展现有 `device_status` 属性，包含详细状态信息

### 3. 实现状态更新机制
- 添加 `update_state` 方法用于更新设备状态
- 实现 UDP 命令接收机制，处理服务器主动推送的状态更新
- 考虑状态更新的频率和时机

### 4. 与现有代码集成
- 确保状态反馈功能与现有控制指令兼容
- 在控制指令执行前后更新状态
- 保持代码结构清晰，易于维护

## 三、代码实现步骤

1. **定义状态数据类**：创建 `DeviceState`、`ReactorState` 和 `PostProcessState` 类
2. **扩展 `UDPClient` 类**：添加接收状态更新的功能
3. **扩展 `ResinWorkstation` 类**：
   - 添加状态属性
   - 实现状态查询方法
   - 实现状态更新方法
   - 扩展 `device_status` 属性
4. **添加状态更新处理逻辑**：处理 UDP 服务器推送的状态更新
5. **测试状态反馈功能**：确保状态查询和更新正常工作

## 四、预期效果

- 能够实时查询设备整体状态
- 能够查询单个反应器的详细状态
- 能够查询后处理系统的状态
- 支持 UDP 服务器主动推送状态更新
- 状态信息包含所有关键参数
- 与现有控制指令无缝集成
